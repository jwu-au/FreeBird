# v3.6.1 更新内容（watch 日志降噪 + 默认轮询调整）

- 🤫 **`fb watch` 不再刷屏「Previous cycle still running」** —— 这条「上个周期还在跑」的提示本来是设计成「提醒一次就安静」的，但之前计数器在每个周期结束时都会清零，导致在持续繁忙（一首接一首地解码大文件）时几乎每次轮询都重新冒一条警告。现在这条提示在**整段繁忙期只出现一次**，等真正处理完积压、闲下来之后，下次再堵才会重新提醒。
- ⏱️ **默认轮询间隔 `5秒` → `30秒`** —— 对于大文件库（尤其是 `.ncm` / FLAC 配合 L3 完整性校验），单个文件解码远超 5 秒，5 秒的默认轮询太频繁、只会一直撞上「上个周期还在跑」。30 秒是更合理的默认值；需要的话仍可用 `--poll-interval` 自行调整。

# v3.6.0 更新内容（新增 .ncm 下载文件支持）

- 🎵 **现在能转换「下载」的歌曲了（`.ncm` 文件）** —— 以前 FreeBird 只能处理在线**试听**时产生的 `.uc` / `.uc!` 缓存文件；现在你**下载**歌曲生成的 `.ncm` 文件也能直接转换了。`fb scan` 和 `fb watch` 会自动识别这两种文件，命令不变、无需加任何参数。
- 📡 **`.ncm` 完全离线解码，不联网** —— 和缓存文件不同，`.ncm` 文件本身就内嵌了歌名、歌手、专辑和封面，所以转换时**不会请求网易云接口**，没网也能转，又快又稳。
  - 🏷️ **自动命名 + 标签**：直接读取文件内嵌的歌名/歌手/专辑，输出命名为 `歌手 - 歌名.flac`（多位歌手用 ` & ` 连接）；万一文件里没有这些信息，则回退使用原文件名。
  - 🖼️ **自动写入专辑封面**：`.ncm` 里自带的封面会一并写进输出文件（FLAC 用 `metaflac` 写入并保留 PCM-MD5，MP3/M4A 用 TagLibSharp）。加上 `--no-write-tags` 可同时关闭标签和封面。
  - 🛡️ **和 `.uc` 一样安全可靠**：原子写入、完整性校验（`off` / `l1` / `l3` / `auto`）、损坏文件自动隔离并附带说明文件；`fb watch` 借助转换记录确保每个 `.ncm` 只转一次、之后自动跳过。**全程绝不修改你的原始文件。**

# 历史版本回顾

> 下面按大版本概括主要里程碑，细节请查阅英文 [`CHANGELOG.md`](../CHANGELOG.md)。

## 🟢 v3.x 系列 —— 从「能用」到「省心」

这一系列是 FreeBird 的成熟阶段，重点是**元数据、标签、稳定性和后台运行**。

- **智能命名与标签**：输出文件不再是冷冰冰的 `歌曲ID.flac`，而是自动从网易云抓取信息，命名为 `歌手 - 歌名.flac`，并写入标题/歌手/专辑标签。可用 `--offline` 保留旧的纯 ID 命名。
- **多文件夹支持**：`fb scan`、`fb watch` 都支持一次传入多个输入文件夹，全部转到同一个输出目录。
- **更聪明的元数据获取**：网络不好或被网易云限流时会自动按指数退避重试，等网络恢复后补上正确歌名；并能正确区分「限流」和「真的查无此歌」，避免误判。
- **可靠性修复**：重启 / 开机后不再对**所有**已转换文件重复请求接口（杜绝「API 风暴」与一屏报错）；离线临时命名的 `歌曲ID.flac` 在补全为正式文件后会被自动清理。
- **Windows 后台服务模式**：`fb service` 命令族，可把 FreeBird 注册成开机自启的 Windows 服务，配合 JSON 配置文件、Windows 事件日志和每日滚动日志，听完歌自动转换、完全不用管。
- **Windows FLAC 校验开箱即用**：需要时自动从官方源下载 `flac` 工具，无需手动安装。
- **开源协议改为 MIT**，README 面向普通用户重写。

## 🔵 v2.0 —— 持续监听模式

- 新增 `fb watch` 子命令：持续盯着缓存目录，有新文件就自动转换。
- 用文件系统中的「旁车记录」标记永久失败的文件，避免反复重试。

## ⚪ v1.0 —— 初版

- 一次性批量解码 `.uc` / `.uc!` 文件（XOR `0xA3`）。
- 按魔术字节识别格式（MP3 / FLAC / M4A）。
- 四级音频完整性校验（`auto` / `l1` / `l3` / `off`）。
- 通过「暂存 + 重命名」实现原子写入。
- 失败文件自动隔离，并附带 `.txt` 说明文件。
